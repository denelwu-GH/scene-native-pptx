# Constrained SVG Profile

## Allowed Elements

`svg`, `defs`, `g`, `rect`, `circle`, `ellipse`, `line`, `path`, `polyline`, `polygon`, `text`, `tspan`, `linearGradient`, `radialGradient`, `stop`, and the controlled shadow filter primitives supported by the bundled converter.

Allow `image` only in `hybrid-fidelity` mode, only with a local relative path declared by the design contract, and only as PNG, JPEG, or WebP. Convert EMF/WMF/TIFF/PDF assets before scene authoring.

## Forbidden Elements

`foreignObject`, `mask`, `textPath`, `animate`, `animateTransform`, `script`, `style`, remote image URLs, remote fonts, and external stylesheets.

Do not use `<use>` in final SVG. Expand reusable icons before validation.

## Attributes

- Use inline presentation attributes; do not rely on CSS classes.
- Use explicit fill, stroke, stroke-width, opacity, font-family, font-size, font-weight, and text-anchor.
- Use `translate` and `scale` transforms only. Avoid matrix, skew, and nested rotation transforms.
- Use stable `id` attributes on top-level groups and major objects.
- Keep gradient stops to three or fewer.
- Keep paths finite and within the viewBox.
- Use relative local paths for hybrid images.

## Text

- Use `<text>` and `<tspan>` only.
- Keep letter spacing at zero.
- Avoid inherited font styles that are not explicit on the text node.
- Preserve exact contract text, punctuation, and line grouping.

## PowerPoint Safety

- Require zero skipped top-level groups in conversion trace.
- Avoid unsupported filters and clipping constructs even when a browser renders them correctly.
- Prefer native shadows generated by the converter over raster glow effects.
- Save and reopen the final deck in Microsoft PowerPoint before delivery.
