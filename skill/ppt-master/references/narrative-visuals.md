# Narrative Visual Workflow

Use this route for user stories, service journeys, comics, and scene-based explanation.

## Build A Visual Bible

Define before generating panels:

- characters: age range, face, hair, clothing, accessories, body language;
- vehicle or product: model class, color, interior, persistent details;
- environment: place, time, weather, cultural context;
- camera: framing, lens feel, viewpoint, and continuity rules;
- lighting and palette;
- prohibited changes and artifacts.

Generate a text-free reference sheet when character or object continuity matters.

## Separate Artwork From Text

Generate scenes without headlines, Chinese labels, data, logos, UI copy, or decorative pseudo-text. Add approved text later as native PowerPoint objects. This keeps text accurate, editable, accessible, and easy to localize.

## Story Contract

Each narrative page or panel sequence must define:

- protagonist and context;
- trigger and need;
- observable actions;
- service or system response;
- user confirmation points;
- fallback, cancellation, or recovery when relevant;
- emotional memory point;
- claim maturity.

Reject magical behavior, implausible task bundling, or a story that implies unverified autonomous authority. Key reservations, payments, itinerary changes, and refunds should show user confirmation unless the source explicitly supports another policy.

## QA

Check continuity across panels, hands and object interaction, vehicle geometry, repeated faces, cultural detail, crop safety, empty space reserved for native text, and whether the scene still communicates without generated writing.
