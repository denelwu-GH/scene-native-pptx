# Launch Post: English

## Headline

Beautiful in. Editable out. Scene Native PPTX is now public.

## Post

Most image-to-PowerPoint workflows force a compromise: preserve the design as a flat image, or rebuild it manually and lose visual fidelity.

Scene Native PPTX takes a different route. It uses a design contract and `scene.json` to rebuild the meaningful parts of a slide as native PowerPoint objects:

- editable text, cards, connectors, paths, gradients, and groups;
- isolated replaceable artwork only where native DrawingML is not the right tool;
- deterministic SVG-to-DrawingML conversion with structural and package validation;
- PowerPoint open/save/reopen validation, not just a screenshot preview.

The public samples currently verify:

- all-native: 85 shapes, 31 editable text runs, 0 pictures;
- hybrid-fidelity: 53 shapes, 22 editable text runs, 1 replaceable artwork asset.

The project is MIT licensed and built as an installable Codex skill.

GitHub: https://github.com/denelwu-GH/scene-native-pptx

Keep the design. Keep the editability. Ship a PowerPoint that survives PowerPoint.

## Short Version

Scene Native PPTX is an open-source Codex skill for rebuilding polished slide designs as stable, native, editable PowerPoint files. It keeps text, cards, connectors, and geometry editable instead of flattening a slide into images. MIT licensed: https://github.com/denelwu-GH/scene-native-pptx
